# 실전 리플레이 데이터 가이드

## 목적과 분리 원칙

`/replay`의 **실전 리플레이**는 고정 일봉을 한 봉씩 공개하며 매매 연습을 하는 별도 화면이다.
대상은 AAPL, MSFT, NVDA, AMZN, GOOGL, META, TSLA, AMD, NFLX, JPM이며 가격은 USD cent 정수로
전달된다. 이 종목들은 기존 대시보드의 KABU·MOCK·NEKO·SAKU·TANU와 완전히 별개다.

리플레이 API는 읽기 전용이다. `market.symbols`, Redis 호가창, `matching.trades`, 주문, 계좌,
정산 테이블에는 리플레이 종목·주문·체결을 추가하지 않는다. 기간·속도·시나리오를 바꿔도 기존
모의거래의 가격, 봇, 보유 수량, 잔고에는 영향이 없다.

## 고정 데이터와 기간

리플레이는 외부 API, 로컬 CSV, 오프라인 fixture를 선택하지 않는다. 종목별로 재현 가능한 고정
연습 경로 1,095개를 내장하며, API 키 유무·네트워크 상태·API 재시작에 따라 결과가 바뀌지 않는다.
이 데이터는 실제 시세 피드나 투자 판단용 데이터가 아니다.

기간은 달력의 영업일 수가 아니라 **정확한 봉 수**를 뜻한다. 따라서 주말·휴일 때문에 1개월에
20여 개 봉만 보이는 문제가 없다.

| 값 | 화면 표시 | 정확한 일봉 수 |
| --- | --- | ---: |
| `1mo` | 1개월 | 30 |
| `3mo` | 3개월 | 90 |
| `6mo` | 6개월 | 180 |
| `1y` | 1년 | 365 |
| `2y` | 2년 | 730 |
| `3y` | 3년 | 1,095 |

`3y`가 최대 기간이다. API는 항상 선택한 기간의 최신 N개 봉을 정확히 반환한다.

## 종목 추가 방식

현재 카탈로그는 다음 10개 종목을 제공한다.

| 거래소 | 종목 |
| --- | --- |
| NASDAQ | AAPL, MSFT, NVDA, AMZN, GOOGL, META, TSLA, AMD, NFLX |
| NYSE | JPM |

추가 종목은 `apps/api/src/replay/replay.catalog.ts`에 카탈로그 항목을,
`apps/api/src/replay/fixed-replay-data.ts`에 동일 id의 고정 프로필을 함께 추가한다. 프로필이 없으면
API가 의도적으로 오류를 내므로, 종목 목록과 고정 데이터가 어긋난 채 배포되지 않는다.

## Alpha Vantage API 키 발급·보관 가이드

현재 리플레이 화면은 고정 데이터를 사용하므로 `ALPHA_VANTAGE_API_KEY`를 읽거나 외부 요청을
보내지 않는다. 아래 절차는 나중에 고정 데이터를 갱신하는 별도 작업을 준비할 때의 키 발급·보관
기준이다. 키를 설정했다고 현재 리플레이 결과가 바뀌지는 않는다.

1. [Alpha Vantage의 API 키 발급 페이지](https://www.alphavantage.co/support/#api-key)를 열고 용도와 조직, 실제로 접근 가능한 이메일을 입력한다.
2. 발급된 키를 복사한다. Alpha Vantage는 키 분실 안내 등에 이메일을 사용한다고 밝히며, 발급·사용에는 이용 약관 동의가 포함된다.
3. 저장소 루트의 로컬 `.env`에만 다음처럼 넣는다. `.env`는 커밋하지 않는다.

   ```dotenv
   ALPHA_VANTAGE_API_KEY=발급받은_개인_키
   ```

4. 키를 `NEXT_PUBLIC_*`, 브라우저 코드, 스크린샷, URL 쿼리, 로그에 넣지 않는다. 별도 데이터 갱신 도구를 도입할 때에도 서버 프로세스에서만 읽도록 한다.
5. 실제 데이터 갱신을 설계하기 전 [공식 API 문서](https://www.alphavantage.co/documentation/)와 약관, 요청 한도 및 해당 엔드포인트의 데이터 권한을 확인한다. 무료 키의 요청량·엔드포인트 범위는 변경될 수 있다.

공식 지원 페이지는 무료 키 발급과 무료 서비스의 일일 요청 한도를 안내한다. 키 또는 수집한 데이터를
재배포할 수 있는 권한을 이 프로젝트가 대신 부여하지는 않는다.

## 기준 경로와 봇 혼합 모드

| 모드 | 가격 경로 | 기존 봇·거래소와의 관계 |
| --- | --- | --- |
| 기준 경로 | 선택한 고정 OHLCV 경로를 그대로 재생 | 봇 없음. `apps/bots`나 실호가창을 읽거나 바꾸지 않음 |
| 봇 혼합 | 기준 경로에 seed 기반의 가상 압력을 더함 | 가상 MM/모멘텀 압력은 리플레이 엔진 안에서만 계산되며 기존 봇 계정과 무관 |

혼합 모드의 가상 가격은 선택한 한도(±1%, ±2.5%, ±5%; 기본 ±5%) 안에 항상 제한된다. 이 한도는
기준 가격에 대한 상한·하한이며 실제 `apps/bots` 프로세스나 Redis 주문장에 주문을 넣는 기능이
아니다. 같은 데이터·시나리오 seed·설정이면 같은 가상 압력이 재현된다.

리플레이 화면의 초기 USD 100,000 가상 계좌와 현재가 모의 매수/매도도 브라우저의 연습 상태다.
실제 로그인 계정의 주문 API를 호출하지 않으므로 대시보드의 체결, 잔고, 기존 5개 종목 호가창에
영향을 주지 않는다.

## API 상태와 문제 해결

- `GET /replay/datasets`는 고정 카탈로그와 최대 1,095봉 정보를 돌려준다.
- `GET /replay/datasets/:datasetId/candles?range=<기간>`은 선택 기간의 정확한 봉 수를 돌려준다.
- `400 Bad Request`: `range`가 `1mo`, `3mo`, `6mo`, `1y`, `2y`, `3y` 중 하나가 아닐 때다.
- `404 Not Found`: 리플레이 카탈로그에 없는 dataset id를 요청한 경우다.

리플레이는 투자 조언이나 실제 주문 기능이 아니다. 외부 배포 또는 장기 보존을 계획한다면 데이터
권리, 제공자 약관, 기업행동 처리, 통화·가격 단위, 감사·보존 정책을 별도로 설계해야 한다.
